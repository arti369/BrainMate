문제 정의

오프라인 강의를 들을 때 집중력, 이해력 문제로 한 번 들어서는 완벽히 강의 내용을 습득할 수 없는 불편한 점이 있습니다. 이러한 문제를 해결하기 위해 AI를 활용한 STT, 내용 요약, 정리 기능이 있는 웹사이트를 만들었습니다. 또한 음성녹음파일의 개인정보 유출, 저작권 침해 문제를 해결하기 위한 설계를 했습니다.

아키텍처/설계

  • '클라우드 API'대신 '로컬 서버' 실행, STT 모델: 'OpenAI Whisper' 채택
  • 요약 모델: 'Hugging Face Transformers' 생태계 활용
  • 워크플로우: '2단계 비동기 API' 분리
  • 데이터 보안: '원본 음성 파일 즉시 삭제' 정책

핵심 기능

온프레미스 STT 엔진

사용자가 업로드한 오디오 파일(회의록, 강의, 통화 녹음 등)을 외부 API로 전송하지 않고 서버 내에 직접 설치된 AI 모델(Whisper)을 통해 텍스트로 변환합니다. 민감한 음성 데이터의 유출을 원천적으로 차단하여 완벽한 데이터 보안을 보장합니다.

AI-인간 협업 편집기

AI가 생성한 텍스트 초안을 사용자가 실시간으로 검토하고 수정할 수 있는 웹 기반 편집기입니다. 고유명사, 전문 용어, 화자별 오류 등을 사용자가 직접 바로잡아 100% 정확한 텍스트 원본을 확보합니다. 이는 AI 요약의 품질을 극대화하는 핵심 단계입니다.

컨텍스트 기반 요약기

사용자의 '수정'을 거친 최종 확정된 텍스트를 기반으로, 서버 내 AI 요약 모델이 작동합니다. 단순 키워드 나열이 아닌, 전체 문맥을 이해하여 핵심 주제, 결정 사항, 실행 항목을 추출하는 지능형 요약문을 생성합니다.

원본 데이터 즉시 삭제

프로세스의 '보안'을 완성하는 핵심 작동 방식입니다. 텍스트 변환이 완료된 원본 오디오 파일은 서버에서 즉시 물리적으로 삭제됩니다.

나의 역할 및 기여

  • API 엔드포인트 설계
  • 웹 디자인
  • 모델 설치 및 환경 설정

결과 및 성과

개인정보, 저작권 침해에서 자유로운 간단한 음성파일 요약, 정리 웹사이트 구축

회고 (배운 점)

  • AI 모델 서빙 및 통합: 모델 로딩 시간, 메모리 관리, 추론 속도 등 실제 서비스 환경에서 부딪히는 성능 문제를 직접 체험하고 고민하게 됨
  • 웹 서버와 긴 작업 처리: 비동기(Asynchronous) 처리의 필요성을 깨닫게 됨
  • 보안 및 파일 처리: 사용자가 업로드한 파일을 안전하게 처리하는 방법(파일명 보안 처리, 임시 저장)을 배움

회고 (개선점)

비동기 작업 큐(Task Queue) 도입

  • 문제점: 지금 구조로는 10분짜리 음성 파일을 올리면, 변환이 끝날 때까지 (몇 분간) 서버가 응답 불능 상태가 되고 사용자는 무한 대기하게 됨
  • 개선: Celery와 Redis (또는 RabbitMQ) 같은 메시지 큐 시스템을 도입

고기능 에디터 도입

  • 문제점: 기본 <textarea>는 단순 텍스트 수정만 가능
  • 개선: Draft.js나 Quill.js 같은 자바스크립트 에디터 라이브러리를 도입하여, 위에서 언급한 '타임스탬프', '화자' 정보를 시각적으로 표시하고 더 편리하게 편집하도록 지원
